Per assegurar les dades d’entrada, també es demana crear una funció validar_adn que, donada una seqüència, validi que només contingui els caràcters vàlids: “A”, “C”, “G” i “T”. D’aquesta manera, si es detecta algun caràcter diferent, es mostra un missatge d’error i no es processa aquesta seqüència.
validar_adn <- function(seq) {
    if (grepl("^[ACGT]+$", seq)) {
        return(TRUE)
    } else {
        return(FALSE)
    }
}

DNA_to_RNA <- function(dna_seq) {
    if (validar_adn(dna_seq)) {
        rna_seq <- gsub("T", "U", dna_seq)
        return(rna_seq)
    } else {
        stop("Error: La seqüència d'ADN no és vàlida.")
    }
}

Exercici 2. Càlcul de l’índex de normalització d’expressió. (1.25 punts)

Suposem un experiment de RNA-seq, a partir del qual volem normalitzar l’expressió d’un gen respecte al total de lectures de la mostra. Aquest ajust de les dades es realitza per tal que siguin comparables.

Suposem que es defineix l’índex de normalització com:

índex = lectures_gen / lectures_totals

Cal tenir en compte que aquests valors han de ser positius i numèrics.

Resoleu les següents qüestions:

  1. (0.5 punts) Definir una funció en R anomenada indice_normalizacion que rebi dos paràmetres:
lecturas_gen: nombre de lectures del gen.
lecturas_totales: nombre total de lectures de la mostra.

La funció ha de calcular i retornar el valor de l’índex, així com realitzar el control d’errors corresponent.

indice_normalizacion <- function(lecturas_gen, lecturas_totales) {
    if (!is.numeric(lecturas_gen)) {
        stop("Error: El nombre de lectures del gen ha de ser numèric.")
    }
    if (!is.numeric(lecturas_totales)) {
        stop("Error: El nombre total de lectures ha de ser numèric.")
    }
    if (lecturas_gen < 0) {
        stop("Error: El nombre de lectures del gen ha de ser positiu.")
    }
    if (lecturas_totales <= 0) {
        stop("Error: El nombre total de lectures ha de ser major que zero.")
    }
    indice <- lecturas_gen / lecturas_totales
    return(indice)
}

Exercici 4. Temps d’eliminació d’un fàrmac en sang (2.5 punts)

L’objectiu d’aquest exercici és estudiar el temps que tarda un fàrmac a reduir la seva concentració en sang després de la seva administració.

Suposem que el temps, en hores, segueix una distribució log-normal i, en condicions normals, la mitjana meanlog és de 1.0 i la desviació típica sdlog de 0.25.

S’ha detectat que els valors d’aquests paràmetres en pacients amb insuficiència renal serien:

meanlog = 1.4
sdlog = 0.35

Nota.- La distribució log-normal és més adequada per tractar temps biològics, creixement bacterià, etc.

Es demana:

Exercici 5. Resposta a un tractament analgèsic en dolor crònic (2.5 punts)

Cal avaluar l’eficàcia d’un nou tractament analgèsic en pacients amb dolor crònic, del qual sabem que no tots els pacients responen igual.

A partir d’estudis previs, el 25% dels pacients “responen al tractament”, és a dir, experimenten una reducció significativa del dolor. En canvi, el 75% restant “no responen al tractament”.

En pacients que responen al tractament, la probabilitat de presentar millora clínica després d’una setmana és del 80%. En la resta de pacients, la probabilitat de presentar millora (efecte placebo o variabilitat) és del 20%.

  1. (0.5 punts) Si seleccionem 120 pacients tractats, quina és la probabilitat que almenys 50 reportin millora? Calcular-ho de forma exacta i mitjançant simulació.
set.seed(123) # Per reproducibilitat
# Paràmetres
n_pacients <- 120
prob_resposta <- 0.25
prob_millora_resposta <- 0.80
prob_millora_no_resposta <- 0.20
# Simulació
simulacions <- 10000
resultats <- replicate(simulacions, {
    pacients <- rbinom(n_pacients, 1, prob_resposta)
    millora <- ifelse(
        pacients == 1,
        rbinom(n_pacients, 1, prob_millora_resposta),
        rbinom(n_pacients, 1, prob_millora_no_resposta)
    )
    sum(millora)
})
# Probabilitat que almenys 50 reportin millora
prob_almenys_50 <- mean(resultats >= 50)
print(paste("Probabilitat que almenys 50 reportin millora (simulació):", prob_almenys_50))
## [1] "Probabilitat que almenys 50 reportin millora (simulació): 0.0769"
